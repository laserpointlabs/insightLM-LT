## How It Works: The Ontology Question

**"How does the tool know how to design an initial architecture?"**

**It doesn't know. The expert knows. Here's the process:**

1. **Capture Expert Knowledge** → Define domain concepts, relationships, and rules (ontology)
2. **Ingest Source Material** → Parse requirements, standards, legacy designs into a knowledge graph
3. **AI Reasons Over Structure** → Suggests components, identifies gaps, flags conflicts
4. **Expert Validates & Refines** → Accepts, modifies, or rejects — the system learns from corrections

> *Think of it as a brilliant junior engineer who reads fast but needs senior judgment. The ontology teaches the vocabulary; the expert provides wisdom.*
